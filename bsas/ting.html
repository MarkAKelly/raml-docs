URI Parameters

  • nino: required(string)

    National Insurance number, in the format AA999999A.

    Example:

    TC663795B

Annotations

  • annotations.scope
    "write:self-assessment"

Headers

  • Accept: required(string)

    Specifies the response format and the version of the API to be used.

    Example:

    application/vnd.hmrc.1.0+json
  • Content-Type: required(string)

    Specifies the format of the request body, which must be JSON.

    Example:

    application/json
  • Gov-Test-Scenario: (string)

    Only in sandbox environment. See Test Data table for all header values.

    Example:

    -

Query Parameters

  • taxYear: required(string)

    The tax year the data applies to, for example, 2019-20. The start year and end year must not span two tax years. No gaps are allowed, for example, 2019-21 is not valid. The minimum tax year is 2019-20.

    Example:

    2019-20
  • typeOfBusiness: (string)

    The type of business the summary calculation is for. Limited to the following possible values:

    • self-employment
    • uk-property-fhl
    • uk-property-non-fhl

    Example:

    self-employment
  • selfEmploymentId: (string)

    An identifier for the self-employment business, unique to the customer. Must conform to the regular expression

    • ^X[A-Z0-9]{1}IS[0-9]{11}$

    Example:

    XAIS12345678901

URI Parameters

    National Insurance number, in the format AA999999A.

    Example:

    TC663795B
  • bsasId: required(string)

    A unique identifier for the summary calculation.

    Must conform to the regular expression ^[0-9]{8}|[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$

    Specifies the response format and the version of the API to be used.

    Example:

    application/vnd.hmrc.1.0+json
  • Content-Type: required(string)

    Specifies the format of the request body, which must be JSON.

    Example:

    application/json
  • Gov-Test-Scenario: (string)

    Only in sandbox environment. See Test Data table for all header values.

    Example:

    -

Query Parameters

  • adjustedStatus: (boolean)

    To specify which of the original - unadjusted - summary or the adjusted summary is to be returned. If not specified the adjusted summary will be returned, where it exists. The original summary is returned, otherwise.

    Example:

    true
